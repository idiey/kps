## Project Overview

**KPS (Sistem Pengurusan Koperasi)** is a multi-site FELDA cooperative management system built with Laravel 12 + Vue 3 + Inertia.js. Core domains:
- **Peneroka** ‚Äî cooperative member management
- **Hutang** ‚Äî debt/loan tracking per peneroka
- **Potongan** ‚Äî monthly salary deductions entry (single or bulk)
- **Allocation** ‚Äî deterministic waterfall: potongan auto-split across debts by priority
- **Month Closing** ‚Äî lock a month's allocation from further edits
- **Multi-site** ‚Äî HQ oversight + site-scoped staff access

Domain glossary: Peneroka = member, Hutang = debt, Potongan = monthly deduction, Pengagihan = allocation.

## Commands

```bash

composer setup              # install deps, migrate, build assets

# Development
composer dev                # Laravel server + queue + Vite (concurrently)
composer dev:ssr            # Same + SSR + log tailing

# Testing
composer test               # Clear config cache, run all Pest tests
php artisan test tests/Feature/KpsSiteTest.php   # Run a single test file

# Code quality
composer format             # PHP: Laravel Pint
npm run lint                # JS/TS: ESLint (auto-fix)
npm run format              # JS/TS: Prettier
npm run format:check        # Check Prettier without fixing

# Frontend build
npm run dev                 # Vite dev server (port 5173)
npm run build               # Production build
```

## Architecture

### Backend (Laravel 12, PHP 8.2+)

**Key directories:**
- `app/Http/Controllers/Kps/` ‚Äî all KPS request handlers
- `app/Models/Kps/` ‚Äî Eloquent models (Site, Peneroka, Debt, MonthlyDeduction, DeductionAllocation, AuditLog)
- `app/Services/Kps/` ‚Äî business logic (AllocationService, MonthlyClosingService, SiteExperienceService)
- `routes/kps.php` ‚Äî all KPS routes, grouped under `kps.` name prefix

**Routing pattern:** All KPS routes are prefixed `/kps`, authenticated, and wrapped in `EnsureKpsSiteContext` middleware. Site-scoped routes follow `/kps/sites/{site}/resource`.

**Multi-site access control:** `EnsureKpsSiteContext` middleware auto-redirects non-admins to their first assigned site. Admins have global access. Site context comes from the `{site}` route parameter.

**Permissions:** Spatie Laravel Permission. Roles: `pentadbiran` (global admin), `company_admin` (HQ), `site_admin`, `staff`. Permissions defined in `RolePermissionSeeder`.

**Allocation waterfall:** `AllocationService` distributes a monthly deduction across a peneroka's debts in priority order. The `unallocated_amount` field on `MonthlyDeduction` tracks leftover. Each manual reallocation is written to `kps_audit_logs`.

### Frontend (Vue 3 + Inertia.js + TypeScript)

**Key directories:**
- `resources/js/pages/Kps/` ‚Äî Vue page components (Inertia auto-discovers by filename)
- `resources/js/components/kps/` ‚Äî KPS-specific reusable components
- `resources/js/routes/kps/` ‚Äî TypeScript route helpers (generated by Laravel Wayfinder)
- `resources/js/actions/` ‚Äî TypeScript types for server controller actions
- `resources/js/layouts/kps/KpsShellLayout.vue` ‚Äî main KPS layout wrapper

**UI stack:** Tailwind CSS 4, Reka UI (headless components), Lucide Vue Next icons.

**Inertia pattern:** Controllers return `Inertia::render('Kps/PageName', [...props])`. Vue pages receive props directly‚Äîno separate API calls needed.

**Route helpers:** Use Wayfinder-generated helpers from `resources/js/routes/kps/` for type-safe route URLs. Don't hardcode URL strings.

### Database

All KPS models use UUID primary keys. Key tables: `sites`, `penerokas`, `debts`, `monthly_deductions`, `deduction_allocations`, `kps_site_user` (pivot with role), `kps_audit_logs`. Tests use in-memory SQLite (`:memory:`).

### Testing

Pest 4 with Laravel plugin. Feature tests hit a real in-memory database‚Äîno mocking of DB. `tests/Pest.php` has global helpers: `ensureRole($name)`.

Tests are organized by feature, not by controller. When writing tests, follow the existing patterns in `tests/Feature/KpsSiteTest.php` and `tests/Feature/KpsAllocationWorkflowTest.php`.

## Key Conventions

- **Controllers stay thin** ‚Äî business logic lives in `Services/Kps/`
- **All KPS routes** must go through `EnsureKpsSiteContext` ‚Äî never bypass it
- **Audit logging** is required for any allocation change ‚Äî use `AuditLog::record()`
- **Month closing** is irreversible ‚Äî the `is_closed` flag on `MonthlyDeduction` gates all writes
- **Wayfinder routes** regenerate automatically on `npm run build` ‚Äî don't hand-edit generated files in `resources/js/routes/`
