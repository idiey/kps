# Getting Started

Use this chapter to sign in, reach the correct KPS workspace, and understand why different users
see different menus.

## Sign In

1. Open the KPS application URL provided by your organization.
2. Enter your email address and password.
3. Complete email verification or two-factor authentication if the system asks for it.
4. After login, the system sends you to the KPS dashboard or your assigned site.

If login fails, check that your account is active. An admin with User Management access can
activate or deactivate accounts.

## Main Navigation

KPS uses role-aware navigation. You only see pages your account can access.

Common areas:

| Area | Purpose |
| --- | --- |
| `Dashboard` | HQ or site overview depending on your access. |
| `Analytics` | HQ analytics across KPS sites. |
| `Sites` | List, view, create, or edit KPS sites where permitted. |
| `Peneroka` | Manage or view peneroka profiles for a selected site. |
| `Hutang` | Manage or view debt records for a selected site. |
| `Potongan Bulanan` | Enter and review monthly deductions. |
| `Allocation Review` | Inspect allocation results, recalculate open deductions, and close a month where permitted. |
| `Reports` | View site reports and peneroka statements. |
| `Audit Trail` | Review logged site actions. |
| `User Management` | Admin-only user administration. |
| `Role Management` | Admin-only role and permission administration. |

## Site Context

Many KPS pages are site-scoped. A site context means the page is working with one selected
site only.

- HQ admins can open `Sites`, choose a site, and switch between site workspaces.
- Site admins and staff are redirected to their first assigned site when they open KPS without a site selected.
- Users without access to a site receive a forbidden page or do not see the site in navigation.

Always confirm the site name or code in the page header before entering `Peneroka`, `Hutang`, or
`Potongan Bulanan` data.

## Admin And Staff Categories

This manual groups roles into two categories:

| Manual Category | Included Roles |
| --- | --- |
| Admin | `pentadbiran`, `company_admin`, `site_admin` |
| Staff | `staff` |

Admin pages and actions are still permission-based. For example, `pentadbiran` can manage users
and roles, while `site_admin` focuses on assigned-site operations.

## Profile Settings

Use profile settings to keep your account details current.

1. Open `Profile Settings` or the KPS profile page.
2. Update your name or email address if the fields are available.
3. Save the form.
4. If your email becomes unverified, use the verification link flow shown by the system.

Profile settings do not change your KPS role, site assignment, or permissions. Ask an admin if
your access needs to change.
